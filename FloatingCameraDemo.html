Floating Camera Component

UdaanIQ Interview Platform - Fixed findDOMNode Error Implementation

Problem Statement RESOLVED

Runtime TypeError: _reactDom.default.findDOMNode is not a function thrown from DraggableCore.findDOMNode. This happens because react-draggable is trying to use ReactDOM.findDOMNode() which is not available/allowed in the app environment.

Root Cause

react-draggable traditionally used findDOMNode() to locate the DOM node. In React 18 and Next.js with strict mode, this API is deprecated and throws errors.

Solution Implemented

Updated the FloatingCamera component to use react-draggable's nodeRef API, which avoids calling ReactDOM.findDOMNode entirely.

Implementation Details COMPLETED

Key Changes Made

  • Added nodeRef to FloatingCamera component to avoid findDOMNode calls
  • Passed nodeRef to the Draggable component
  • Attached the same nodeRef to the DOM element
